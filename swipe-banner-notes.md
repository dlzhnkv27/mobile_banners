# Swipe Banner Fix Notes

## Проблема

При горизонтальном свайпе баннера страница микроскроллилась по вертикали (дергалась вверх/вниз), потому что:
- pull-to-refresh стартовал даже при таче на баннере;
- свайп баннера не блокировал нативный вертикальный скролл браузера на раннем этапе жеста.

## Внесенные изменения

1. **Исключен pull-to-refresh для тача на баннере**
   - В `ptrDown` добавлена проверка `e.target.closest('#bannersSection')`.
   - Если жест начат в зоне баннеров, pull-to-refresh не активируется.

2. **Добавлен direction-lock для свайпа баннера**
   - Для жеста баннера введены `startY` и `dragAxis` (`x`/`y`).
   - На `touchmove`:
     - если жест вертикальный (`dragAxis === 'y'`) — карусель не перетаскивается;
     - если жест горизонтальный (`dragAxis === 'x'`) — вызывается `preventDefault()` для подавления микроподскролла страницы.

3. **Включена возможность `preventDefault` в `touchmove`**
   - Слушатель `document.addEventListener('touchmove', onMov, { passive: false })`.

## Ожидаемый результат

- Горизонтальный свайп баннеров работает без вертикального дергания страницы.
- Вертикальный скролл страницы сохраняется при обычных вертикальных жестах.
